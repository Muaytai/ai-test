# AI Doctor — анализ родинок и меланом по фото

Веб-приложение с нейросетью для **предварительной** оценки кожных образований по фотографии: доброкачественная родинка или подозрительное образование (возможная меланома).

**Важно:** это не медицинский диагноз. Результат носит информационный характер и не заменяет осмотр дерматолога или онколога.

## Команды (run.py)

| Команда | Описание |
|---------|----------|
| `python run.py` | Запуск веб-сервера (по умолчанию http://localhost:8000) |
| `python run.py download` | Скачать HAM10000 с Kaggle и разложить в data/benign, data/malignant |
| `python run.py train` | Обучить модель (требуются данные в data/benign и data/malignant) |

Параметры для `download`: `--data-dir`, `--limit` (макс. снимков на класс). Для `train`: `--data-dir`, `--epochs`, `--batch-size`, `--weights-dir`, `--img-size` — см. раздел «Как дообучить модель».

## Как запустить

1. Установите зависимости (рекомендуется виртуальное окружение):

```bash
python -m venv .venv
.venv\Scripts\activate   # Windows
# source .venv/bin/activate  # Linux/macOS
pip install -r requirements.txt
```

2. Переменные окружения через `.env`:
   - Скопируйте `cp .env.example .env` (или вручную создайте `.env` из примера).
   - В `.env` укажите при необходимости `KAGGLE_API_TOKEN`, `HOST`, `PORT`. Файл `.env` в git не попадает.

3. Запустите сервер:

```bash
python run.py
```

4. Откройте в браузере: **http://localhost:8000** (или тот host:port, что заданы в `.env`)

Загрузите фото кожного образования — модель вернёт оценку и рекомендацию.

## Данные для обучения

### Скачать HAM10000 и разложить по папкам (одной командой)

Нужен аккаунт [Kaggle](https://www.kaggle.com/) и API-токен:

1. **Токен:** [Kaggle → Settings → API](https://www.kaggle.com/settings) → **Create New Token**. Скопируйте токен (показывается один раз).
2. В файле **`.env`** (скопируйте из `.env.example`) укажите:  
   `KAGGLE_API_TOKEN=KGAT_ваш_токен`  
   При запуске `python run.py download` переменные подхватываются из `.env`.
3. На странице датасета [Skin Cancer MNIST: HAM10000](https://www.kaggle.com/datasets/kmader/skin-cancer-mnist-ham10000) нажмите **Join** / примите правила.
4. Из корня проекта выполните:

```bash
python run.py download
```

Снимки скачаются с Kaggle и автоматически разложатся в **data/benign/** и **data/malignant/** (по полю `dx`: меланома и BCC → malignant, остальное → benign). Опции: `--data-dir` (папка для данных, по умолчанию `data`), `--limit N` (макс. снимков на класс, для теста): `python run.py download --limit 500`.

### Если не используете Kaggle

Положите свои изображения вручную в **data/benign/** и **data/malignant/** (форматы .jpg, .png). Или скачайте [HAM10000](https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/DBW86T) / [ISIC](https://www.isic-archive.com/) и разложите по диагнозу.

## Модель и процесс обучения

### Какая модель используется

- **База:** [EfficientNetB0](https://arxiv.org/abs/1905.11946) из Keras Applications — свёрточная сеть, предобученная на **ImageNet** (1.4M изображений, 1000 классов). Используется как экстрактор признаков: веса базы **заморожены** (не обучаются), дообучается только верхняя часть.
- **Вход:** изображение **224×224** пикселей, RGB, нормализация в диапазон [0, 1].
- **Верх модели (голова):** глобальный средний пул (GAP) → Dense(128, ReLU) → Dropout(0.4) → Dense(1, sigmoid). Выход — вероятность класса «злокачественное» (меланома/BCC).
- **Классы:** бинарная классификация — **benign** (доброкачественное) и **malignant** (злокачественное: меланома, базально-клеточный рак).
- **Фреймворк:** TensorFlow 2 / Keras. Оптимизатор: Adam. Функция потерь: binary cross-entropy. Метрики: accuracy, AUC.

### Как проходит обучение

1. **Данные:** изображения читаются из папок `data/benign/` и `data/malignant/` через `ImageDataGenerator`. **20%** выборки откладывается на валидацию (`validation_split=0.2`).
2. **Аугментация (только для обучения):** случайные повороты (±20°), сдвиги по ширине/высоте (20%), отражения по горизонтали и вертикали. Нормализация в [0, 1] (`rescale=1/255`).
3. **Эпохи:** на каждой эпохе модель проходит по всем батчам обучающей выборки, считает loss и метрики; затем оценивает **val_loss** и **val_auc** на валидации.
4. **Сохранение:** сохраняются веса с **наилучшим val_auc** (ModelCheckpoint). Если val_auc не улучшается **3 эпохи подряд**, обучение останавливается (EarlyStopping), восстанавливаются лучшие веса.
5. **Вывод:** в консоль выводится прогресс по эпохам (loss, auc, val_auc) и ориентировочное время до конца обучения.

После обучения файл весов попадает в `app/model_weights/melanoma_model.weights.h5`. При запуске приложения модель подгружает эти веса и по новому фото возвращает вероятность «злокачественное» и текстовую рекомендацию.

## Как дообучить модель

Модель по умолчанию — EfficientNetB0 (веса ImageNet) + бинарная «голова». После появления данных в **data/benign/** и **data/malignant/** запустите обучение.

### Запуск обучения

Из корня проекта (с активированным venv):

```bash
python run.py train --data-dir data --epochs 15
```

Параметры:

| Параметр       | По умолчанию | Описание |
|----------------|--------------|----------|
| `--data-dir`   | `data`       | Папка с подпапками `benign` и `malignant` |
| `--epochs`     | 10           | Число эпох |
| `--batch-size` | 32           | Размер батча (больше = быстрее эпоха, если хватает памяти) |
| `--weights-dir`| `app/model_weights` | Куда сохранить веса |
| `--img-size`   | 224          | Сторона изображения; 192 — быстрее, чуть хуже качество |

Пример: `python run.py train --data-dir D:\datasets\skin_lesions --epochs 20 --batch-size 16`

**Ускорение обучения:** 1) Обучайте на ПК с **GPU** (NVIDIA) — в разы быстрее. 2) Увеличьте `--batch-size` (32–64), если хватает ОЗУ/VRAM. 3) На CPU можно попробовать `--img-size 192` (тогда в `app/config.py` задайте `IMG_SIZE=192` и для инференса).

Обучение использует 20% данных для валидации, аугментации (повороты, сдвиги, отражения) и сохраняет лучшие веса по `val_auc`. При переобучении сработает early stopping.

### После обучения

Веса сохраняются в `app/model_weights/melanoma_model.weights.h5`. При следующем запуске `python run.py` приложение подхватит их автоматически — перезапустите сервер и загружайте фото в интерфейсе.

## API

- **POST /api/analyze** — загрузка изображения (формат `multipart/form-data`, поле `file`). Ответ в JSON:
  - `class_name`: `"benign"` или `"malignant"`
  - `confidence`: уверенность модели (0–1)
  - `probability_malignant`: вероятность «злокачественно»
  - `is_risk`: `true` при подозрении на меланому
  - `message`: текст рекомендации для пользователя

## Стек

- **Frontend:** HTML/CSS/JS, загрузка файла и отображение результата
